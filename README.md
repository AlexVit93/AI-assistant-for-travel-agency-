# AI-ассистент для автоматической квалификации клиентов туристического агентства

Данный проект — интеллектуальный AI-ассистент, разработанный для автоматизации первичной обработки обращений клиентов туристического агентства.

В отличие от обычных чат-ботов, которые лишь собирают контактные данные, система выполняет полноценную квалификацию клиента, собирает информацию о будущем путешествии, структурирует её и автоматически передает менеджерам в готовом для работы виде.

## Основная задача проекта

Сократить время менеджеров на первичное общение с клиентами и автоматизировать сбор заявок на подбор туров.

После завершения диалога менеджеру не требуется дополнительно выяснять базовые параметры поездки. Вся необходимая информация уже собрана, обработана и сохранена системой.

---

## Как работает система

### 1. Первичное общение

AI-ассистент начинает диалог с клиентом и выступает в роли виртуального менеджера туристического агентства.

В процессе общения система уточняет:

- страну или направление отдыха;
- количество взрослых;
- наличие и возраст детей;
- желаемые даты путешествия;
- ориентировочный бюджет;
- требования к отдыху;
- пожелания по размещению;
- дополнительные условия поездки.

### 2. Квалификация клиента

Ассистент определяет уровень заинтересованности клиента и помогает перевести диалог к оформлению полноценной заявки.

При необходимости обрабатываются возражения и предлагаются альтернативные способы связи.

### 3. Сбор контактной информации

После получения параметров поездки система запрашивает:

- имя клиента;
- номер телефона;
- удобный способ связи.

### 4. Структурирование данных с помощью GPT

После завершения диалога модель GPT анализирует всю историю переписки и преобразует её в структурированную заявку.

Пример автоматически сформированной карточки клиента:

```json
{
  "name": "Максим",
  "phone_number": "+375291234567",
  "trip_direction": "Турция",
  "people_number": "2 взрослых",
  "travel_dates": "Август 2025",
  "budget": "2000$",
  "customer_wishes": "Первая линия, всё включено"
}
```

### 5. Автоматическое сохранение

Полученные данные одновременно сохраняются:

- в PostgreSQL;
- в Google Sheets;
- в журнал диалогов JSON.

### 6. Передача менеджеру

Менеджер получает уже готовую квалифицированную заявку и может сразу перейти к подбору конкретных туров и отелей, не тратя время на сбор базовой информации.

## Возможности проекта

- AI-консультирование клиентов.
- Сбор параметров будущей поездки.
- Автоматическая квалификация лидов.
- Извлечение структурированных данных из диалога.
- Сохранение заявок в PostgreSQL.
- Выгрузка лидов в Google Sheets.
- Хранение истории общения.
- Поддержка длинного контекста переписки.
- Работа по заранее заданным бизнес-сценариям туристического агентства.

## Практическая ценность

Проект позволяет заменить менеджера на этапе первичного общения с клиентом и передавать сотрудникам уже готовые, квалифицированные заявки.

В результате менеджер не занимается сбором информации вручную, а сразу приступает к подбору туристических предложений на основании данных, собранных системой.
